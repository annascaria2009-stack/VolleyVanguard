# VolleyVanguard Website — Design

**Date:** 2026-05-21
**Author:** Anna Scaria (with Claude)
**Status:** Approved for planning

## Overview

A public showcase website for **VolleyVanguard**, a volleyball coaching organization founded by Anna Scaria and friends (a team of 5–6 coaches). The site introduces the organization, attracts new players/clients, displays the training schedule, and shows photo galleries of coaches and events.

The defining constraints:

- **Free to host**, indefinitely.
- A **non-technical admin** (Anna) must be able to update all text and upload/replace photos later, through a friendly visual dashboard — never touching code.
- For now the site ships with **sample placeholder copy and free, no-copyright placeholder images**, to be replaced via the dashboard.

## Goals

1. Showcase who VolleyVanguard is and introduce the coaches.
2. Attract players/parents/teams and make it easy to reach out.
3. Display the schedule with **contact-based registration** (inquire → enroll manually). No online payments or accounts.
4. Share updates/news and event results over time.
5. Photo galleries of coaches and coaching events.

## Non-Goals (YAGNI)

- Online booking system, user accounts, or payments.
- Multi-language support.
- Comments, forums, or social features.
- Any custom backend or database.

## Architecture

A **static website** generated by **Astro**, with content stored as editable data files in a **GitHub** repository and edited through a visual CMS. Hosting auto-rebuilds on every save.

```
Admin (Anna)
   │  edits text / uploads photos in a friendly form
   ▼
/admin  ── Sveltia CMS (visual editor, runs in browser) ── GitHub OAuth login
   │  saves → commits content files
   ▼
GitHub repo  (markdown / JSON content + images)
   │  push triggers build
   ▼
Netlify  ── runs `astro build` → deploys static site (free)
   ▼
Public website  (fast, static HTML/CSS/JS)
```

### Components

- **Astro static site** — builds all pages to static HTML/CSS. Chosen for excellent content-site support, image optimization, and simple component model.
- **Sveltia CMS** (Decap-compatible) at `/admin` — a single-page visual editor. Anna logs in with a free GitHub account (one time), then edits content in forms and uploads photos by clicking. Chosen over classic Decap CMS for its modern, reliable GitHub authentication and better media/UX.
- **GitHub repository** — stores site code + all editable content (markdown/JSON) and uploaded images. Doubles as version history/backup.
- **Netlify** (free tier) — hosts the site, rebuilds automatically on each commit, and provides **GitHub OAuth** for the CMS login (registered GitHub OAuth app; Netlify handles the token handshake — no separate auth server to maintain). *Alternative: Cloudflare Pages + a small auth Worker; Netlify chosen for the simplest free auth path.*
- **Netlify Forms** (free) — powers the contact/inquiry form; submissions arrive by email and in the Netlify dashboard. No backend code.

## Pages

Separate pages sharing a common nav bar and footer:

| Page | Contents |
|------|----------|
| **Home** | Hero (large photo bg + tagline + CTA), short intro, highlight stats, coaches preview, gallery strip, "join us" CTA band |
| **About** | VolleyVanguard's story and mission |
| **Coaches** | Grid of coach cards — large photo, name, role, bio (5–6 coaches) |
| **Programs** | The coaching services / programs offered |
| **Schedule** | Schedule table + "Contact us to register" button |
| **Gallery** | Larger photo grid of coaches and coaching events |
| **News** | Announcements / event results feed (newest first) |
| **Contact** | Inquiry form (Netlify Forms) + email/WhatsApp/Instagram links |

## Content Model (editable in the CMS)

- **Site settings** — org name, tagline, logo, contact email, WhatsApp number, Instagram URL, footer text.
- **Home page** — hero heading, hero subtext, hero image, intro paragraph, highlight stats.
- **About page** — heading + body text, optional image.
- **Coaches** — collection: name, role, photo, bio, display order.
- **Programs** — collection: title, description, image/icon.
- **Schedule** — collection of rows: day, time, program, level, location.
- **Gallery** — collection: image, caption, category (coach / event).
- **News** — collection: title, date, cover image, body.

Each collection supports add / edit / reorder / delete and photo upload from the dashboard.

## Visual Design

**Bold Athletic** direction (selected):

- **Colors:** deep navy (`#0b1f3a` / `#13315c`), energetic coral accent (`#ff5a3c`), white / light-grey surfaces (`#f1f5fb`).
- **Type:** strong bold display headings (e.g. Anton or Archivo Black via Google Fonts) + clean body font (Inter).
- **Imagery:** **large, prominent photos** — full-width hero, tall coach cards, generously sized gallery tiles. Photos are a primary design element, not accents.
- **Tone:** confident, sporty, serious-but-welcoming.
- **Responsive:** mobile-first; must look great on phones (most parents/players will browse on mobile).
